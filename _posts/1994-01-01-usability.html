World Wild Web

Unlike Eric Mayer "The King of CSS", Jakob Nielsen a Danish researcher later coined as "The King of Usability" has a very public so {% include atoms/link.html url="http://www.dourish.com/nielsen.html" title="Hacking Jakob Nielsen" text="controversial" %} profile.

While other pioneers were working on the web infrastructure—the browser, structuring (HTML) and styling (CSS) content— Nielsen started to think with the readers and users mind.

In the first years the World Wild Web looks funny; there were no web designers, web developers, user experience engineers but sysadmins and webmasters. Mostly with Computer Science background these gatekeepers of the web often came up with ... strange visual solutions.

Human-Computer Interaction

In this scene appears a young scientist fluent in Human-Computer Interactions with strong support from an IT giant, Sun Microsystems.

You're supposed to be the world's leading expert in your field, so you figure out what would be most important for the company for you to work on.

{% include atoms/link.html url="http://www.nngroup.com/people/jakob-nielsen/" title="Jacob Nielsen's Bio" text="Jacob Nielsen's Bio" %}

Nielsen started to research what makes websites work, how to deliver the message in this shiny new medium. He was not focused on graphic design—typography, layout, colors, icons, etc.— but influencing visistor's behaviour in such way to achieve a goal like clicking a button which, for example, has the 'Purchase' text on it.

Many of his results are de-facto standard in today's web user interface design: {% include atoms/link.html url="http://www.nngroup.com/articles/f-shaped-pattern-reading-web-content/" title="F-Shaped Pattern For Reading Web Content" text="F-Shaped Pattern For Reading Web Content" %}, {% include atoms/link.html url="http://www.nngroup.com/articles/113-design-guidelines-homepage-usability/" title="Design Guidelines for Homepage Usability" text="Design Guidelines for Homepage Usability" %} and {% include atoms/link.html url="http://www.nngroup.com/articles/ten-usability-heuristics/" title="10 Usability Heuristics for User Interface Design" text="10 Usability Heuristics for User Interface Design" %}.

He concludes visitors can focus only on five things on a single page and identifies these focus points with their function:

focus point function
1 identity makes the visitor feeling safe, being in the right place
2 offer what this page offers
3 values emphasize what your site does that's valuable from the user's point of view, as well as how you differ from key competitors.
4 action emphasize the highest priority tasks so that users have a clear starting point for what to do next.
5 entertainment if all above are not interesting for the user make it feel good by offering something fun which still connects / links back to your site.

Well designed pages today follows exactly these rules. You don't have to wonder why sometimes you are getting excited to add things in your shopping cart.
